# Python Version Management

## Current Version: 3.12.7

This project uses Python 3.12.7 as specified in:
- `.python-version` (for pyenv)
- `runtime.txt` (for Heroku)

## Using pyenv (Recommended)

### Why pyenv?
- Manage multiple Python versions on your system
- Automatically switch to the correct version for each project
- Avoid "works on my machine" issues
- Match production environment exactly

### Setup pyenv

**macOS/Linux:**
```bash
# Install pyenv
curl https://pyenv.run | bash

# Add to shell configuration (~/.bashrc, ~/.zshrc, etc.)
export PATH="$HOME/.pyenv/bin:$PATH"
eval "$(pyenv init --path)"
eval "$(pyenv init -)"

# Restart shell or source config
source ~/.bashrc  # or ~/.zshrc

# Install Python 3.12.7
pyenv install 3.12.7

# Set as local version for this project
cd /path/to/SpookyOctober
pyenv local 3.12.7

# Verify
python --version  # Should show: Python 3.12.7
```

**Windows:**
```powershell
# Install pyenv-win using PowerShell
Invoke-WebRequest -UseBasicParsing -Uri "https://raw.githubusercontent.com/pyenv-win/pyenv-win/master/pyenv-win/install-pyenv-win.ps1" -OutFile "./install-pyenv-win.ps1"; &"./install-pyenv-win.ps1"

# Install Python 3.12.7
pyenv install 3.12.7

# Set as local version
pyenv local 3.12.7

# Verify
python --version
```

### How .python-version Works

The `.python-version` file tells pyenv which Python version to use for this project.

When you `cd` into the SpookyOctober directory:
1. pyenv reads `.python-version`
2. Automatically switches to Python 3.12.7
3. All `python` commands use the correct version

No need to manually activate/deactivate!

## Without pyenv

If you don't want to use pyenv:

1. **Download Python 3.12.7** from [python.org](https://www.python.org/downloads/)
2. **Install it** (make sure to check "Add to PATH")
3. **Create virtual environment** with that specific version:
   ```bash
   python3.12 -m venv .venv
   source .venv/bin/activate
   ```

## Checking Your Python Version

```bash
# Check Python version
python --version

# Check pip version
pip --version

# Check if in virtual environment
which python  # Should point to .venv/bin/python
```

## Common Issues

### "python: command not found"
- Python is not installed or not in PATH
- Try `python3` instead of `python`

### "pyenv: version `3.12.7` is not installed"
- Run: `pyenv install 3.12.7`

### Wrong Python version showing
- Make sure you're in the project directory
- Run: `pyenv local 3.12.7` again
- Restart your shell

### Virtual environment using wrong Python
- Delete `.venv`: `rm -rf .venv`
- Recreate with correct version: `python -m venv .venv`

## Updating Python Version

If we need to upgrade Python in the future:

1. Update `.python-version`: `echo "3.13.0" > .python-version`
2. Update `runtime.txt`: `echo "python-3.13.0" > runtime.txt`
3. Install new version: `pyenv install 3.13.0`
4. Recreate virtual environment:
   ```bash
   rm -rf .venv
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
